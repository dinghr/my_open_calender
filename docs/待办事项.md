### 3. 🔑 创建 Secrets 配置文件
**状态：** ⏳ 待执行  
**为什么需要：** 存储敏感信息（API 密钥、管理员密码、JWT 密钥），不能提交到 Git  
**预计时间：** 5 分钟

**需要做什么：**
```bash
# 生产环境
sudo nano /etc/ai-study/prod.secrets

# 测试环境
sudo nano /etc/ai-study/test.secrets
```

**填入内容（参考模板）：**
```bash
# /etc/ai-study/prod.secrets

# AI API 密钥（从 Moonshot 获取）
AI_API_KEY=sk-your-actual-api-key-here

# 管理员账号
ADMIN_USERNAME=admin
ADMIN_PASSWORD=YourSecurePassword123!

# JWT 密钥（生成随机密钥）
JWT_SECRET=$(openssl rand -hex 32)
# 将上面命令输出的密钥填入此处
```

**设置权限：**
```bash
sudo chmod 600 /etc/ai-study/prod.secrets
sudo chmod 600 /etc/ai-study/test.secrets
```

**对应脚本模板：**
- `deploy/prod.secrets.example`
- `deploy/test.secrets.example`

**为什么不能自动执行：**
- 包含敏感信息（API 密钥、密码）
- 需要你自己设置强密码
- 需要你自己从 AI 服务商获取 API 密钥

```bash
# 查看 SQLite 数据库
sqlite3 data/dev.db

# 执行 SQL 查询
sqlite3 data/dev.db "SELECT * FROM students;"

# 进入交互模式
sqlite3 data/dev.db
sqlite> .tables
sqlite> SELECT * FROM students;
sqlite> .quit
```

# 查看服务状态
