## 🔍 新增表结构详解

### 1. user_student_relations (账号关系表)

```sql
CREATE TABLE user_student_relations (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL,           -- 用户ID
    student_id UUID NOT NULL,        -- 小朋友ID
    relation VARCHAR(30),            -- 关系：father, mother, grandfather等
    status VARCHAR(20) DEFAULT 'pending', -- pending/approved/rejected
    created_at TIMESTAMP,
    approved_at TIMESTAMP,           -- 审批时间
    approved_by UUID,                -- 审批人
    UNIQUE(user_id, student_id)      -- 同一用户对同一小朋友只能有一条关系
);
```

**关系类型：**
- `father` - 父亲
- `mother` - 母亲
- `grandfather` - 爷爷
- `grandmother` - 奶奶
- `maternal_grandfather` - 外公
- `maternal_grandmother` - 外婆
- `other` - 其他

### 1.1 student_invites (关联邀请表) ⭐ 新增

```sql
CREATE TABLE student_invites (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL,
    created_by UUID NOT NULL,
    invite_code VARCHAR(20) UNIQUE NOT NULL,  -- 邀请码/链接 token
    invite_type VARCHAR(20) DEFAULT 'qr',     -- qr/link
    status VARCHAR(20) DEFAULT 'active',      -- active/used/expired/disabled
    max_uses INTEGER DEFAULT 1,
    current_uses INTEGER DEFAULT 0,
    expires_at TIMESTAMP,
    created_at TIMESTAMP,
    used_at TIMESTAMP
);
```

### 2. spinosaurus_records (棘龙记录表)

```sql
CREATE TABLE spinosaurus_records (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL UNIQUE,
    level INTEGER DEFAULT 1,         -- 等级 1-10
    attack INTEGER DEFAULT 100,      -- 攻击力
    defense INTEGER DEFAULT 50,      -- 防御力
    experience INTEGER DEFAULT 0,    -- 经验值
    streak_days INTEGER DEFAULT 0,   -- 连续打卡天数
    max_streak_days INTEGER DEFAULT 0, -- 最长连续天数
    last_active_date DATE,           -- 最后活跃日期
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

**等级经验对照表：**
| 等级 | 所需经验 | 攻击力基础 | 防御力基础 |
|-----|---------|----------|----------|
| Lv.1 | 0 | 100 | 50 |
| Lv.2 | 100 | 120 | 60 |
| Lv.3 | 300 | 150 | 75 |
| ... | ... | ... | ... |
| Lv.10 | 10000 | 500 | 250 |

### 3. points_records (积分记录表)

```sql
CREATE TABLE points_records (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL,
    points INTEGER NOT NULL,         -- 正数为获得，负数为消耗
    source VARCHAR(50) NOT NULL,     -- 来源类型
    source_id UUID,                  -- 关联记录ID
    balance INTEGER NOT NULL,        -- 变动后余额
    description VARCHAR(255),
    created_at TIMESTAMP
);
```

**积分来源类型：**
- `task_complete` - 任务完成
- `reading_complete` - 阅读完成
- `reading_bonus` - 阅读奖励（按时/提前）
- `review_submit` - 读后感提交
- `reward_exchange` - 奖励兑换（负数）
- `streak_bonus` - 连续打卡奖励

### 4. reading_records (阅读记录表)

```sql
CREATE TABLE reading_records (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL,
    book_id UUID NOT NULL,
    status VARCHAR(20) DEFAULT 'reading', -- reading/completed/abandoned
    progress INTEGER DEFAULT 0,      -- 进度 0-100
    words_read INTEGER DEFAULT 0,    -- 已读字数
    start_date DATE,
    expected_end_date DATE,          -- 预计完成日期
    completed_at TIMESTAMP,
    review_text TEXT,                -- 读后感内容
    rating INTEGER,                  -- 评分 1-5
    points_earned INTEGER DEFAULT 0, -- 获得积分
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    UNIQUE(student_id, book_id)
);
```

### 5. bookshelf (书架管理表)

```sql
CREATE TABLE bookshelf (
    id UUID PRIMARY KEY,
    student_id UUID NOT NULL,
    book_id UUID NOT NULL,
    is_recommended BOOLEAN DEFAULT FALSE, -- 推荐给书友
    is_lendable BOOLEAN DEFAULT FALSE,    -- 可借阅（必须先推荐）
    lend_status VARCHAR(20) DEFAULT 'available', -- available/lent
    lent_to_student_id UUID,         -- 借给谁
    added_at TIMESTAMP,
    UNIQUE(student_id, book_id)
);
```

**业务规则：**
- `is_lendable = TRUE` 前提是 `is_recommended = TRUE`
- 借出时 `lend_status = 'lent'`，归还后恢复为 `'available'`

### 6. book_reviews (读后感评论表)

```sql
CREATE TABLE book_reviews (

    parent_review_id UUID,           -- 回复的评论ID（支持嵌套）
    likes_count INTEGER DEFAULT 0,
    created_at TIMESTAMP,
    is_deleted BOOLEAN DEFAULT FALSE
);
```

### 7. rewards (奖励表)

```sql
CREATE TABLE rewards (
    id UUID PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description VARCHAR(255),
    icon VARCHAR(50),                -- emoji 或图标名
    points_cost INTEGER NOT NULL,    -- 积分消耗
    required_level VARCHAR(20),      -- star/moon/sun
    is_active BOOLEAN DEFAULT TRUE,
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP
);
```

**等级限制：**
- `star` - 星星等级可用（0-700积分）
- `moon` - 月亮等级解锁（700-2000积分）
- `sun` - 太阳等级解锁（2000+积分）

## 📈 数据统计视图

### 1. 学生阅读统计视图

```sql
CREATE VIEW student_reading_stats AS
SELECT
    s.id AS student_id,
    s.nickname,
    COUNT(DISTINCT CASE WHEN rr.status = 'completed' THEN rr.book_id END) AS books_completed,
    COALESCE(SUM(CASE WHEN rr.status = 'completed' THEN rr.words_read ELSE 0 END), 0) AS total_words,
    COALESCE(SUM(CASE WHEN rr.status = 'completed' THEN rr.points_earned ELSE 0 END), 0) AS total_points
FROM students s
LEFT JOIN reading_records rr ON s.id = rr.student_id
GROUP BY s.id, s.nickname;
```

### 2. 书友排行榜视图

```sql
CREATE VIEW book_friend_ranking AS
SELECT
    s.id AS student_id,
    s.nickname,
    s.avatar_url,
    SUM(rr.words_read) AS total_words,
    COUNT(DISTINCT CASE WHEN rr.status = 'completed' THEN rr.book_id END) AS books_count,
    RANK() OVER (ORDER BY SUM(rr.words_read) DESC) AS rank
FROM students s
JOIN reading_records rr ON s.id = rr.student_id
WHERE rr.status = 'completed'
GROUP BY s.id, s.nickname, s.avatar_url;
```

### 3. 积分余额视图

```sql
CREATE VIEW student_points_balance AS
SELECT
    s.id AS student_id,
    s.nickname,
    COALESCE(SUM(pr.points), 0) AS balance,
    CASE
        WHEN COALESCE(SUM(pr.points), 0) >= 2000 THEN 'sun'
        WHEN COALESCE(SUM(pr.points), 0) >= 700 THEN 'moon'
        ELSE 'star'
    END AS level
FROM students s
LEFT JOIN points_records pr ON s.id = pr.student_id
GROUP BY s.id, s.nickname;
```

## 📊 数据迁移脚本

### 从 v1.0 迁移到 v2.0

```sql
-- 1. 添加学生新字段
ALTER TABLE students ADD COLUMN IF NOT EXISTS age INTEGER;
ALTER TABLE students ADD COLUMN IF NOT EXISTS gender VARCHAR(10);
ALTER TABLE students ADD COLUMN IF NOT EXISTS textbook_version VARCHAR(50);
ALTER TABLE students ADD COLUMN IF NOT EXISTS hobbies JSON;

-- 2. 创建棘龙记录（为现有学生初始化）
INSERT INTO spinosaurus_records (student_id, level, attack, defense)
SELECT id, 1, 100, 50 FROM students
ON CONFLICT (student_id) DO NOTHING;

-- 3. 创建账号关系（将现有家长绑定到学生）
INSERT INTO user_student_relations (user_id, student_id, relation, status, approved_at)
SELECT
    u.id,
    s.id,
    'parent',
    'approved',
    NOW()
FROM users u
JOIN students s ON s.parent_id = u.id
WHERE u.role = 'parent';
```

## 🔐 数据隐私保护

### 敏感数据处理

| 数据类型 | 存储方式 | 访问权限 |
|---------|---------|---------|
| 用户手机号 | 加密存储 | 仅本人和管理员 |
| 家长密码 | 哈希存储 | 仅验证时使用 |
| 儿童个人信息 | 明文存储 | 仅绑定家长 |
| 错题原始图片 | 本地文件/OSS 可选 | 仅本人和家长 |
| 共享题目来源 | 匿名化 | 不显示原学生信息 |

### 数据备份策略

- **全量备份：** 每日凌晨 2:00
- **增量备份：** 可选（按需）
- **备份保留：** 30 天滚动
- **异地备份：** 阿里云 OSS 异地存储（可选）
