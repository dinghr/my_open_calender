## 📋 产品概述

### 产品定位
一款面向**家长和学生**的智能伴学产品，通过 AI 技术辅助学习计划管理、错题整理和知识点掌握。

### 目标用户
| 角色 | 使用场景 | 核心诉求 |
|-----|---------|---------|
| 管理员 | 生成登录码、管理用户权限、审批绑定 | 简单、可控、安全 |
| 家长 | 辅导孩子学习、管理学习计划、设置奖励 | 高效、省心、专业 |
| 学生 | 完成学习任务、整理错题、阅读打卡 | 简单、有趣、有成就感 |

### 平台支持
- **微信小程序**（首期，使用静态登录码）
- **App**（iOS/Android，二期）
- **通用后台 API**（统一服务多端）
- **管理后台**（Web 端，用于登录码管理和用户管理）

## 👤 用户角色与权限

### 角色定义

| 角色 | 权限 | 说明 |
|-----|------|-----|
| 管理员 | 登录码管理、用户管理、审批操作 | 家庭中的主管理员，控制访问权限 |
| 家长用户 | 完整权限 | 可管理多个孩子账号，设置奖励 |
| 学生用户 | 受限权限 | 仅查看自己的计划和错题，阅读打卡 |

### 账号绑定规则 ⭐ 新增

| 规则 | 说明 |
|-----|------|
| 单账号绑定上限 | 每个账号最多绑定 **2 位**小朋友 |
| 多人绑定同一位小朋友 | 支持，需管理员审批通过 |
| 关系类型 | 父亲、母亲、爷爷、奶奶、外公、外婆、其他 |
| 审批机制 | 所有登录和绑定操作都需要管理员审批 |

### 账号关联发起方式（二维码/邀请链接）

- 不展示小朋友列表，不提供搜索入口
- 由已绑定的家长生成邀请二维码/链接
- 亲属扫码后自动带上小朋友 ID，仅需选择关系类型并提交申请
- 申请进入待审批状态，由后台统一审核
- 邀请链接支持有效期、使用次数、手动关闭

## 🎯 核心功能

### 1. 登录功能（静态登录码）

| 需求点 | 描述 | 优先级 |
|-------|------|-------|
| 登录码生成 | 管理员在后台生成一次性/可重复使用的登录码 | P0 |
| 登录码验证 | 用户输入登录码完成登录 | P0 |
| 登录码管理 | 管理员可查看/禁用/删除已发放的登录码 | P1 |
| 审批流程 | 新登录码需管理员审批后生效 | P0 |
| 多端统一 | 同一登录码支持小程序 + App 登录 | P1 |

**登录码状态：**
- `待审批`：新创建，等待管理员审批
- `已激活`：审批通过，可正常使用
- `已禁用`：被管理员禁用
- `已过期`：超过有效期

### 8. 计划功能

#### 8.1 学校课表

| 功能 | 描述 |
|-----|------|
| 每日课程 | 展示今日课程安排 |
| 书本准备 | 提示需要带的书籍和材料 |
| 完整课表 | 跳转子页面查看完整周课表 |
| 放学时间 | 显示今日放学时间 |

#### 8.2 课后安排

| 功能 | 描述 |
|-----|------|
| 时间安排 | 放学后到睡前的详细时间表 |
| 任务类型 | 学习、兴趣班、休息、吃饭等 |
| 地点标注 | 如"📍 少年宫" |

#### 8.3 每日一练打印

**每日一练范围：** 语文 + 数学（暂不包含英语）

**语文模块（按推荐权重动态组合）：**
- 古诗推荐
- 易错字练写
- 好句精读

**家长后台反馈：** 语文模块中不认识的字可一键加入生词本。

| 功能 | 描述 | 优先级 |
|-----|------|-------|
| 预览入口 | 语文/数学分开预览 | P0 |
| 确认机制 | 家长需在放学前确认内容 | P0 |
| 打印按钮 | 确认后方可打印 | P0 |
| 分科打印 | 支持语文/数学单独或合并打印 | P0 |

**确认流程：**
1. 家长查看每日一练预览
2. 确认内容无误
3. 确认后解锁打印按钮
4. 选择打印方式（语文/数学/全部）

#### 8.4 家长倾向性调整（自然语言）

**示例输入：**
- "这周重点练习乘法"
- "语文多练阅读理解"
- "最近计算题错得多，多推一些计算题"

#### 8.5 老师每日学习建议 ⭐ 新增

**入口：** 家长专区 → 粘贴老师建议

**流程：**
1. 家长粘贴老师学习建议原文
2. AI 结构化解析（学科、课题、知识点、作业任务、提醒事项）
3. 家长确认解析结果
4. 写入当日学习进度与推荐权重

**作用范围：**
- 影响每日一练推荐排序（按老师建议提升权重）
- 影响错题举一反三范围（优先覆盖老师当日进度）
- 提醒类内容进入待办清单（如带跳绳）

## 📊 数据模型（核心表）

### 新增/修改的核心表

#### 棘龙记录表 (spinosaurus_records) ⭐ 新增
```sql
- id: UUID
- student_id: UUID
- level: Integer (等级 1-10)
- attack: Integer (攻击力)
- defense: Integer (防御力)
- experience: Integer (经验值)
- streak_days: Integer (连续天数)
- created_at: DateTime
- updated_at: DateTime
```

#### 积分记录表 (points_records) ⭐ 新增
```sql
- id: UUID
- student_id: UUID
- points: Integer (积分变动)
- source: String (来源：任务完成/阅读/奖励兑换等)
- balance: Integer (变动后余额)
- created_at: DateTime
```

#### 图书表 (books) ⭐ 新增
```sql
- id: UUID
- title: String
- author: String
- word_count: Integer
- category: String
- description: Text
- cover_url: String
- created_at: DateTime
```

#### 阅读记录表 (reading_records) ⭐ 新增
```sql
- id: UUID
- student_id: UUID
- book_id: UUID
- status: Enum(reading, completed)
- progress: Integer (进度百分比)
- words_read: Integer
- started_at: DateTime
- completed_at: DateTime
```

#### 书架管理表 (bookshelf) ⭐ 新增
```sql
- id: UUID
- student_id: UUID

- is_recommended: Boolean (是否推荐)
```

#### 小朋友表 (students) 修改
```sql
- id: UUID
```
