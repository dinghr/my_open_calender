## 💾 备份策略

### 本地备份
- **频率：** 每天凌晨 2 点
- **位置：** `/home/ecs-user/my_open_calender/backup/`
- **格式：** `app.db.YYYYMMDD_HHMMSS.tar.gz`
- **保留：** 最近 30 天

### 阿里云 OSS 备份（可选）
- **启用方式：** 编辑 `scripts/backup.sh`，设置 `ENABLE_OSS=true`
- **配置 OSS：**
  ```bash
  # 安装 ossutil
  wget https://gosspublic.alicdn.com/ossutil/install.sh
  bash install.sh
  
  # 配置 credentials
  ossutil config
  ```

### 手动备份
```bash
# 随时手动备份
/home/ecs-user/my_open_calender/scripts/backup.sh

# 或简单复制数据库文件
cp /home/ecs-user/my_open_calender/data/app.db /path/to/backup/app.db.backup
```

## ⚠️ 注意事项

1. **SQLite 限制：**
   - 不适合高并发写入（你的场景够用）
   - 单文件最大 140TB（远超你的需求）

2. **备份检查：**
   - 每周检查备份日志
   - 每月测试一次恢复流程

3. **安全性：**
   - 数据库文件权限：`chmod 600 data/app.db`
   - 备份文件加密（可选）

4. **性能优化：**
   - 定期 VACUUM：`sqlite3 data/app.db "VACUUM;"`
   - 添加索引（如按日期查询学习记录）

## 📞 问题排查

### 问题 1：初始化失败
```
❌ 初始化失败：unable to open database file
```
**解决：** 确保 `data/` 目录存在且有写权限
```bash
mkdir -p data
chmod 755 data
```

### 问题 2：备份脚本无权限
```
❌ Permission denied
```
**解决：** 添加执行权限
```bash
chmod +x /home/ecs-user/my_open_calender/scripts/backup.sh
```

### 问题 3：crontab 不执行
**解决：** 检查 cron 服务状态
```bash
systemctl status cron
# 或
service cron status
```
