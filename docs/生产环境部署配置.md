## 🔧 Nginx 配置

### 生产环境 Nginx 配置

**文件：** `/etc/nginx/sites-available/ai-study-prod`

```nginx
# AI 伴学小程序 - 生产环境
server {
    listen 80;
    server_name api.ai-study.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.ai-study.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/api.ai-study.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.ai-study.com/privkey.pem;
    
    # SSL 优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    
    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;

    # 日志
    access_log /var/log/nginx/ai-study-prod-access.log;
    error_log /var/log/nginx/ai-study-prod-error.log;

    # 客户端大小限制
    client_max_body_size 10M;

    # 健康检查（公开）
    location /health {
        proxy_pass http://127.0.0.1:8001/health;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        access_log off;
    }

    # API 接口
    location /api/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
        
        # 限流（可选）
        # limit_req zone=api_limit burst=20 nodelay;
    }

    # API 文档（需认证）
    location /docs {
        auth_basic "API Documentation";
        auth_basic_user_file /etc/nginx/.htpasswd;
        proxy_pass http://127.0.0.1:8001/docs;
        proxy_set_header Host $host;
    }
}
```

## 🚀 Systemd 服务配置

### 生产环境服务

**文件：** `/etc/systemd/system/ai-study-prod.service`

```ini
[Unit]
Description=AI 伴学小程序 API - Production
After=network.target

[Service]
Type=simple
User=ecs-user
Group=ecs-user
WorkingDirectory=/home/ecs-user/my_open_calender/backend

# 环境变量
Environment="PATH=/home/ecs-user/.nvm/versions/node/v24.14.0/bin:/usr/local/bin:/usr/bin:/bin"
Environment="PYTHONPATH=/home/ecs-user/my_open_calender/backend"
Environment="APP_ENV=production"
Environment="DB_PATH=/home/ecs-user/my_open_calender/data/prod.db"
Environment="HOST=127.0.0.1"
Environment="PORT=8001"
Environment="DEBUG=false"
Environment="LOG_LEVEL=error"

# 从 secrets 文件读取敏感信息
EnvironmentFile=/etc/ai-study/prod.secrets

# 启动命令
ExecStart=/home/ecs-user/.nvm/versions/node/v24.14.0/bin/python3 main.py
Restart=always
RestartSec=5

# 资源限制
LimitNOFILE=65535
LimitNPROC=4096

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=ai-study-prod

[Install]
WantedBy=multi-user.target
```

## 🔒 防火墙配置

### UFW 规则

```bash
# 启用 UFW
sudo ufw enable

# 允许 SSH
sudo ufw allow 22/tcp

# 允许 HTTP/HTTPS (生产)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 允许测试 HTTPS
sudo ufw allow 8443/tcp

# 拒绝后端直接访问（只允许 Nginx 访问）
sudo ufw deny 8001/tcp
sudo ufw deny 8002/tcp

# 查看状态
sudo ufw status verbose
```

## 🔧 部署命令

### 1. 安装依赖

```bash
# 安装 Nginx
sudo apt update
sudo apt install -y nginx

# 安装 Certbot（SSL 证书）
sudo apt install -y certbot python3-certbot-nginx

# 申请 SSL 证书
sudo certbot --nginx -d api.ai-study.com
sudo certbot --nginx -d test-api.ai-study.com
```

### 2. 配置 Nginx

```bash
# 创建配置
sudo cp docs/nginx-prod.conf /etc/nginx/sites-available/ai-study-prod
sudo cp docs/nginx-test.conf /etc/nginx/sites-available/ai-study-test

# 启用站点
sudo ln -s /etc/nginx/sites-available/ai-study-prod /etc/nginx/sites-enabled/
sudo ln -s /etc/nginx/sites-available/ai-study-test /etc/nginx/sites-enabled/

# 移除默认站点
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 3. 配置 Systemd 服务

```bash
# 创建 secrets 目录
sudo mkdir -p /etc/ai-study
sudo chmod 700 /etc/ai-study

# 创建 secrets 文件
sudo cp backend/.env.production /etc/ai-study/prod.secrets
sudo chmod 600 /etc/ai-study/prod.secrets

# 创建 systemd 服务
sudo cp deploy/ai-study-prod.service /etc/systemd/system/
sudo cp deploy/ai-study-test.service /etc/systemd/system/

# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl enable ai-study-prod
sudo systemctl start ai-study-prod

sudo systemctl enable ai-study-test
sudo systemctl start ai-study-test
```

### 4. 配置防火墙

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 8443/tcp
sudo ufw enable
```

### 5. 验证部署

```bash
# 检查服务状态
sudo systemctl status ai-study-prod
sudo systemctl status ai-study-test

# 检查 Nginx
sudo systemctl status nginx

# 健康检查
curl https://api.ai-study.com/health
curl https://test-api.ai-study.com:8443/health

# 查看日志
sudo journalctl -u ai-study-prod -f
sudo tail -f /var/log/nginx/ai-study-prod-access.log
```

```
/var/log/nginx/ai-study-*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data adm
    sharedscripts
    postrotate

}
```
