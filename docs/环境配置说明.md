# 环境配置说明

**更新时间：** 2026-03-18

---

## 📋 环境概述

项目现在支持两个独立的环境：

| 环境 | 用途 | 数据库 | API 地址 |
|------|------|--------|----------|
| **dev** | 开发测试 | `data/dev.db` | `http://localhost:8000` |
| **prod** | 生产部署 | `data/prod.db` | `https://api.yourdomain.com` |

---

## 🚀 快速开始

### 开发环境

```bash
# 1. 切换到 dev 分支
git checkout dev

# 2. 初始化开发数据库（首次运行）
python3 scripts/init_dev_db.py

# 3. 启动后端
cd backend
source .env.development && python3 main.py

# 4. 启动前端（新终端）
cd frontend
npm run dev
```

### 生产环境

```bash
# 1. 切换到 prod 分支
git checkout prod

# 2. 初始化生产数据库（首次部署）
python3 scripts/init_prod_db.py

# 3. 配置环境变量
export APP_ENV=production
export DB_PATH=/path/to/prod.db
export ADMIN_USERNAME=your_admin
export ADMIN_PASSWORD=your_secure_password

# 4. 启动服务
cd backend
python3 main.py
```

## 🔧 环境变量

### 后端环境变量

```bash
# 必需
APP_ENV=development|production
DB_PATH=/path/to/database.db

# 可选
DEBUG=true|false
LOG_LEVEL=debug|info|warning|error
ALLOWED_ORIGINS=https://domain1.com,https://domain2.com
ADMIN_USERNAME=admin
ADMIN_PASSWORD=secret
```

### 前端环境变量

```bash
# 必需
VITE_APP_ENV=development|production
VITE_API_BASE_URL=http://localhost:8000/api

# 可选
VITE_ENABLE_DEBUG=true
VITE_LOG_LEVEL=debug
```

## 📊 分支策略

```
main        - 主分支，稳定版本
├── dev     - 开发分支，新功能开发
└── prod    - 生产分支，线上部署
```

### 工作流程

1. **新功能开发：** 在 `dev` 分支开发
2. **测试验证：** 在 dev 环境测试
3. **发布部署：** 合并到 `prod` 分支
4. **紧急修复：** 直接在 `prod` 修复，然后合并回 `dev`

## 🔐 安全注意事项

### 开发环境
- ✅ 可以使用简单的密码
- ✅ 允许 CORS 所有来源
- ✅ 开启调试模式
- ❌ 不要使用真实的用户数据

### 生产环境
- ✅ 使用强密码
- ✅ 限制 CORS 域名
- ✅ 关闭调试模式
- ✅ 使用环境变量管理敏感信息
- ✅ 定期备份数据库

## 🐛 常见问题

### Q: 如何切换环境？
```bash
# 开发环境
export APP_ENV=development
export DB_PATH=data/dev.db

# 生产环境
export APP_ENV=production
export DB_PATH=data/prod.db
```

### Q: 数据库文件在哪里？
- 开发：`data/dev.db`
- 生产：`data/prod.db`

### Q: 如何备份数据库？
```bash
# 开发环境
cp data/dev.db data/dev.db.backup.$(date +%Y%m%d)

# 生产环境
cp data/prod.db data/prod.db.backup.$(date +%Y%m%d)
```
