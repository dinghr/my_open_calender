## 🛠️ 开发建议

### 1. 后端 API 开发顺序

**第一阶段（MVP）：**
1. 登录验证（`/api/login`）
2. 绑定小朋友（`/api/students/bind`）
3. 获取/发布每日计划（`/api/plans`）
4. 完成任务打卡（`/api/tasks/complete`）

**第二阶段：**
5. 每日一练（`/api/daily-practice`）
6. 老师建议（`/api/teacher-suggestions`）
7. 棘龙系统（`/api/spinosaurus`）
8. 阅读记录（`/api/reading`）
9. 生词管理（`/api/vocabulary-words`）

**第三阶段：**
10. 错题管理（`/api/wrong-questions`）
11. 学习报告（`/api/reports`）
12. 家长备课（`/api/parent-prep`）
13. 自然语言调整（`/api/adjustments`）

### 2. 技术选型建议

| 模块 | 推荐技术 |
|-----|---------|
| 后端框架 | FastAPI（已安装） |
| 数据库 | SQLite（已配置） |
| 小程序前端 | 原生小程序 或 Uni-app |
| 部署 | 阿里云 ECS（已有） |
| 备份 | 本地 + 阿里云 OSS（可选） |

### 3. 文件组织建议

```
backend/
├── main.py              # FastAPI 入口
├── models.py            # 数据库模型（已有）
├── schemas.py           # Pydantic 数据验证
├── api/
│   ├── login.py         # 登录接口
│   ├── students.py      # 学生管理
│   ├── plans.py         # 计划管理
│   └── ...
└── utils/
    ├── auth.py          # 认证工具
    └── generator.py     # 题目生成器
```
